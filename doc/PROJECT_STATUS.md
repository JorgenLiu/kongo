- 当前已验证平台：macOS

- 当前 UI 风格：Agenda 暖棕色工作台，Material 3，亮 / 暗色主题

## 已落地的关键能力

### 桌面体验

- 一级导航与次级入口结构稳定
- 桌面端快捷键已接入
- 页面转场已接入
- 主要列表项 hover 快捷操作已接入
- 联系人 / 事件 / 总结 / 标签 / 文件库主要右键菜单已接入
- 核心卡片已补充基础 Semantics 语义

### 日历与重要日期

- 联系人重要日期已具备独立模型与持久化
- 首页和联系人列表页已展示即将到来的重要日期
- 周历 / 月历已支持展示联系人重要日期节点、公共纪念日节点与营销节点
- 设置中心已支持时间节点类别开关，并持久化到 JSON 偏好文件
- 农历节点当前不参与日历展示

### 设置偏好

- 主题模式已支持 JSON 偏好持久化
- 时间节点类别开关已统一接入 JSON 偏好持久化
- 当前设置偏好不再分散为“部分内存态、部分数据库态”

### 文件库

- 已支持 `managed` / `linked` 混合存储元数据
- 已支持预览状态字段与基础预览状态展示
- 已支持按最近更新 / 文件名 / 文件大小排序
- 已支持存储模式过滤与“仅看原文件缺失”组合筛选
- 已支持文件库多选模式与批量删除孤立附件
- 已支持文件深度预览对话框、预览文本展示与手动刷新预览

### 待办

- 已支持独立待办组与组内子项编辑
- 子项当前支持完成状态、截止日期、多联系人 / 多事件关联
- 待办板当前已支持待办组归档、分组范围切换、事项筛选、排序
- 联系人详情页与事件详情页当前已支持相关待办展示和快速创建关联待办
- 当前尚未将总结行动项自动转换为待办项，也未提供批量操作

## 当前最重要的缺口

### 功能缺口

1. **系统提醒与通知闭环** — P0 优先级，提醒数据模型已有字段但 macOS 通知链路未落地
2. **联系人详情附件模块深化** — 当前仍是轻量聚合入口
3. **节气等更多时间节点源** — 农历节气当前不参与日历展示
4. **待办来源整合与自动流转** — 总结行动项 → 待办组的自动流转未打通
5. **设置中心偏好扩展** — 语言、通知策略、导入导出偏好等仍未进入统一设置模型

### 技术债务

1. 测试覆盖率偏向 provider 和 widget 层，repository 层和 model 层测试目录当前为空
