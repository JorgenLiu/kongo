## 已落地的关键能力

### 桌面体验

- 一级导航与次级入口结构稳定
- 桌面端快捷键已接入
- 页面转场已接入
- 主要列表项 hover 快捷操作已接入
- 联系人 / 事件 / 总结 / 标签 / 文件库主要右键菜单已接入
- 核心卡片已补充基础 Semantics 语义

### 日历与重要日期

- 联系人重要日期已具备独立模型与持久化
- 首页和联系人列表页已展示即将到来的重要日期
- 周历 / 月历已支持展示联系人重要日期节点、公共纪念日节点与营销节点
- 设置中心已支持时间节点类别开关，并持久化到 JSON 偏好文件
- 农历节点当前不参与日历展示

### 设置偏好

- 主题模式已支持 JSON 偏好持久化
- 时间节点类别开关已统一接入 JSON 偏好持久化
- AI 提供商、API Key、Base URL、Model 已统一接入 JSON 偏好持久化
- 当前设置偏好不再分散为“部分内存态、部分数据库态”

### AI 基础设施

- 已落地 `AiProvider` / `DefaultAiService` 抽象，具备统一请求与结果落库通路
- 已落地 OpenAI 兼容 HTTP Provider，可对接硅基流动、DeepSeek、通义千问和自定义兼容端点
- 设置中心已提供 AI 配置区块，支持保存配置与实时测试连接
- App 启动阶段已按本地配置加载 Provider，并通过依赖注入向页面树提供 `AiService`
- 首页 AI 日报 V1 技术上已落地（JSON 解析、缓存、降级），但**经 2026-03-28 评估后调整为非首页主路径**：冷启动无数据时 AI 只能复读结构化字段，无增量价值；AI 将用于用户积累足量非结构化记录后的总结与回顾（用户主动触发）
- iOS Keychain / macOS Keychain AI 密钥存储已通过 Swift 桥接落地

### 文件库

- 已支持 `managed` / `linked` 混合存储元数据
- 已支持预览状态字段与基础预览状态展示
- 已支持按最近更新 / 文件名 / 文件大小排序
- 已支持存储模式过滤与“仅看原文件缺失”组合筛选
- 已支持文件库多选模式与批量删除孤立附件
- 已支持文件深度预览对话框、预览文本展示与手动刷新预览

### 待办

- 已支持独立待办组与组内子项编辑
- 子项当前支持完成状态、截止日期、多联系人 / 多事件关联
- 待办板当前已支持待办组归档、分组范围切换、事项筛选、排序
- 联系人详情页与事件详情页当前已支持相关待办展示和快速创建关联待办
- 当前尚未将总结行动项自动转换为待办项，也未提供批量操作

## 当前最重要的缺口

### 功能缺口

1. **菜单栏快捷捕获（P0）** — 核心新入口，macOS NSStatusItem + 全局快捷键 + MethodChannel 桥接，预计 2-3 天实施
2. **Quick Capture 输入解析（P0）** — 联系人库 fuzzy match + 启发式规则识别人名，单步确认新联系人
3. **首页工作台重构（P1）** — 移除 AI 日报卡，改为规则驱动的单张提醒卡（无上下文会议、长时间未联系等触发规则）
4. **AI 能力重新定位** — 不再做冷启动 homepage 简报；改为用户积累足量记录后的摘要/回顾功能（用户主动触发）
5. **系统提醒深化** — P0 优先级，当前已补齐 macOS 默认提醒值、会后补充与双档稍后提醒；更多平台支持、静默时段与通知策略细化仍未完成
3. **联系人详情附件模块深化** — 当前仍是轻量聚合入口
4. **节气等更多时间节点源** — 农历节气当前不参与日历展示
5. **待办来源整合与自动流转** — 总结行动项 → 待办组的自动流转未打通
6. **设置中心偏好扩展** — 语言、通知策略、导入导出偏好等仍未进入统一设置模型
