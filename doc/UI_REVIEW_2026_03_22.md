# Kongo UI/UX 评审记录

**评审日期**：2026-03-22  
**性质**：时点评审记录，不作为项目事实源；项目事实以代码与其余核心文档为准。

## 总结

当前 UI 基线已经具备可持续演进的结构基础：

- 主题 token 已集中到 `config/`
- shell 与主要页面结构已稳定
- screen -> action -> widget 的拆分方向基本成立
- 搜索高亮、列表 / 详情组件和共享状态组件已具备复用性

但桌面端体验仍明显落后于数据层成熟度，主要缺口集中在交互层，而不是数据库或基础架构层。

## 当前优势

1. 主题、颜色、间距、尺寸 token 已集中
2. 联系人、事件、总结、附件、检索主流程已具备明确入口
3. 搜索结果高亮与排序已经落地
4. 页面级 provider 和 read-side 聚合已经建立稳定模式

## 当前主要问题

### P0
1. 缺少全局快捷键
2. 缺少右键菜单
3. 系统提醒能力尚未接入桌面端工作流

### P1
1. 设置页仍更像项目说明页，而不是用户设置中心
2. 文件库功能偏基础，缺排序、筛选、批量操作
3. 联系人详情中的附件入口仍是提示式入口，不是完整模块页

### P1
1. Hover 态覆盖不足
2. 桌面端窗口与导航交互仍可加强
3. 文件与标签未进入一级导航，需要继续验证当前信息架构是否稳定

## 当前建议优先级

1. 先补系统提醒与桌面原生交互
2. 再补文件库和联系人详情模块深化
3. 之后再评估一级导航信息架构是否需要调整

## 备注

本评审已按当前代码状态校准，不再沿用早期蓝青色设计方案或事件状态视图假设。
