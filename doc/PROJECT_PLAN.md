# Kongo 项目规划

## 产品定位

Kongo 不是传统通讯录，而是一个本地优先的信息工作台。

它围绕四类对象组织个人工作信息：

1. 联系人：长期关系背景与标签
2. 事件：具体互动、协作与跟进记录
3. 每日总结：当天推进情况与明日计划
4. 附件：文件证据、资料与沉淀内容

## 当前产品边界

### 已形成闭环的主能力

- 联系人 CRUD、标签、重要日期、详情页聚合
- 事件 CRUD、参与人、事件类型、搜索与过滤、详情页
- 每日总结 CRUD、按日期唯一、Markdown 编写 / 预览、行动项提取
- 附件导入、事件 / 总结关联、文件库总览与搜索
- 全局检索（联系人 / 事件 / 每日总结）
- 桌面端壳导航、快捷键、页面转场、关键列表 hover 快捷操作、主要右键菜单
- 周历 / 月历视图，以及联系人重要日期节点接入日历

### 当前仍未闭环的部分

- 联系人详情中的附件仍是轻量聚合入口，不是独立联系人附件模块
- 文件库已补齐排序、组合筛选、批量选择删除与深度预览；联系人详情中的附件仍是轻量聚合入口
- 系统提醒已在 macOS 上落地到完整闭环 v1 的主要能力：事件提醒、联系人重要日期提醒、会后补充提醒、新建默认提醒值，以及事件类提醒的双档稍后提醒
- 设置中心已成型；主题模式与时间节点类别开关已支持 JSON 偏好持久化，但更完整设置项仍未完成
- 时间节点已接入联系人重要日期、公共纪念日、营销节点，并支持类别开关；节气等更多节点源仍待扩展
- 待办已进入独立模块第一版，并已支持归档、筛选、排序以及联系人 / 事件详情双向联动；总结行动项与待办组之间仍未打通自动流转

## 当前产品决策

### 1. 事件状态不是当前主语义

- Dart 运行时 `Event` 模型不含 `status`
- 事件列表、详情、表单都不再按状态驱动
- 数据库 `events.status` 仅保留为兼容字段

### 2. 总结主模型是 `DailySummary`

- 当前总结按日期组织，而不是按事件组织
- 历史 `event_summaries` 仅用于迁移与兼容

### 3. 标签与文件库仍作为次级入口存在

- 一级导航当前保持：日程、通讯录、检索、待办、总结、设置
- 标签管理与文件库通过次级入口进入

### 4. 时间节点按统一读模型继续扩展

- 当前已经将联系人重要日期、公共纪念日、营销节点接入周历 / 月历
- 当前类别开关已进入设置中心，并与主题模式一起持久化到 JSON 偏好文件
- 后续节点源统一沿“日历时间节点”抽象扩展，而不是在 UI 中写死特判

### 5. 待办已切换为独立数据模型

- 当前待办已支持独立待办组、一级项 / 子项结构
- 子项支持独立完成状态、截止日期，以及多联系人 / 多事件关联
- 当前待办组已支持归档；待办板已支持分组范围、事项筛选与排序
- 联系人详情页 / 事件详情页已支持查看相关待办并直接创建关联待办
- 当前总结中的行动项提取仍保留，但尚未自动流转到待办组

## 近期路线图

详细路线与技术决策见 `PRODUCT_STRATEGY_2026.md`、`PRODUCT_DIRECTION_2026_03_28.md`、`TECH_STACK_DECISIONS_2026_03_28.md`，以下为主要优先级：

### 当前迭代（进行中）

1. **首页复原（P0a）** — 移除首页 AI 日报卡，恢复工作台布局；AI 基础设施保留供后续使用（见 `PLAN_QUICK_CAPTURE_IMPL.md`）
2. **macOS 菜单栏 Quick Capture（P0b）** — NSStatusItem 常驻图标 + 全局快捷键 `⌃⌘K` 唤起输入框；通过 MethodChannel 写入本地 SQLite；核心用户日常入口
3. **Quick Capture 输入解析（P1）** — 联系人库 fuzzy match + 启发式中英文人名识别；单步确认新联系人；全程离线，无 AI 调用

### 短期（P0/P1 完成后）

1. **规则驱动单张提醒卡（P2）** — 替代 AI 日报卡位置；由确定性规则生成（无会议背景、重逢提醒、新联系人未跟进等）；每次只展示 1 张
2. **系统提醒深化** — 在 macOS 闭环 v1 基础上，继续补更多平台支持、静默时段与更细通知策略

### 中期（Quick Capture 稳定后）

1. AI 联系人关系洞察 — 用户积累足够非结构化记录后，在联系人详情页生成跟进提示（用户主动触发）
2. Outlook 日历 + 通讯录接入 — Microsoft Graph OAuth；作为"数据加速器"，不是 onboarding 前置步骤
3. 联系人详情附件模块深化
4. 设置中心偏好扩展（语言、通知策略、导入导出偏好）
5. 节气等更多时间节点源

### 长期

1. 云同步 — macOS + iOS 用 iCloud CloudKit（record-level，非文件同步）；Windows 跨平台用 E2EE 中继（另起计划）
2. iOS 主屏幕 Widget + Share Extension — 依赖 CloudKit 同步先落地；使用 flutter_home_widget
3. Onboarding 引导流程（首次启动引导）
4. Windows / iOS 完整构建与行为验证

## 当前阶段不做的事

以下方向已知有价值，但不在当前阶段主迭代范围内：

1. 多端实时协作（实时协同编辑）
2. 大规模 AI 自动化工作流（无人工确认的全自动写入）
3. 复杂附件解析流水线（OCR / 语义提取管道）
4. 首页 AI 自动日报（冷启动无数据上下文时 AI 只能复读结构化字段，无增量信息量；AI 保留用于数据成熟后的用户主动触发场景）
