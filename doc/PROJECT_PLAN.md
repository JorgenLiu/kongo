# Kongo 项目规划

## 产品定位

Kongo 不是传统通讯录，而是一个本地优先的信息工作台。

它围绕四类对象组织个人工作信息：

1. 联系人：长期关系背景与标签
2. 事件：具体互动、协作与跟进记录
3. 每日总结：当天推进情况与明日计划
4. 附件：文件证据、资料与沉淀内容

## 当前产品边界

### 已形成闭环的主能力

- 联系人 CRUD、标签、重要日期、详情页聚合
- 事件 CRUD、参与人、事件类型、搜索与过滤、详情页
- 每日总结 CRUD、按日期唯一、Markdown 编写 / 预览、行动项提取
- 附件导入、事件 / 总结关联、文件库总览与搜索
- 全局检索（联系人 / 事件 / 每日总结）
- 桌面端壳导航、快捷键、页面转场、关键列表 hover 快捷操作、主要右键菜单
- 周历 / 月历视图，以及联系人重要日期节点接入日历

### 当前仍未闭环的部分

- 联系人详情中的附件仍是轻量聚合入口，不是独立联系人附件模块
- 文件库已补齐排序、组合筛选、批量选择删除与深度预览；联系人详情中的附件仍是轻量聚合入口
- 提醒字段已存在，但系统通知尚未打通
- 设置中心已成型；主题模式与时间节点类别开关已支持 JSON 偏好持久化，但更完整设置项仍未完成
- 时间节点已接入联系人重要日期、公共纪念日、营销节点，并支持类别开关；节气等更多节点源仍待扩展
- 待办已进入独立模块第一版，并已支持归档、筛选、排序以及联系人 / 事件详情双向联动；总结行动项与待办组之间仍未打通自动流转

## 当前产品决策

### 1. 事件状态不是当前主语义

- Dart 运行时 `Event` 模型不含 `status`
- 事件列表、详情、表单都不再按状态驱动
- 数据库 `events.status` 仅保留为兼容字段

### 2. 总结主模型是 `DailySummary`

- 当前总结按日期组织，而不是按事件组织
- 历史 `event_summaries` 仅用于迁移与兼容

### 3. 标签与文件库仍作为次级入口存在

- 一级导航当前保持：日程、通讯录、检索、待办、总结、设置
- 标签管理与文件库通过次级入口进入

### 4. 时间节点按统一读模型继续扩展

- 当前已经将联系人重要日期、公共纪念日、营销节点接入周历 / 月历
- 当前类别开关已进入设置中心，并与主题模式一起持久化到 JSON 偏好文件
- 后续节点源统一沿“日历时间节点”抽象扩展，而不是在 UI 中写死特判

### 5. 待办已切换为独立数据模型

- 当前待办已支持独立待办组、一级项 / 子项结构
- 子项支持独立完成状态、截止日期，以及多联系人 / 多事件关联
- 当前待办组已支持归档；待办板已支持分组范围、事项筛选与排序
- 联系人详情页 / 事件详情页已支持查看相关待办并直接创建关联待办
- 当前总结中的行动项提取仍保留，但尚未自动流转到待办组

## 近期路线图

详细路线与技术决策见 `PRODUCT_STRATEGY_2026.md`，以下为主要优先级：

### 当前迭代（进行中）

1. **Schema v9 迁移** — 为 8 张核心表添加 `deletedAt` 软删除字段（见 `PLAN_SCHEMA_MIGRATION_V9.md`）
2. **AI 基础设施通路** — `OpenAiCompatibleProvider` + 设置页 AI 配置区块（见 `PLAN_AI_INFRASTRUCTURE.md`）
3. **首页重构** — 信息层级优化：今日日程主卡 + 本周概况 + 待办提醒层（见 `HOME_UI_UX_DETAIL_PLAN_2026_03_26.md`）

### 短期（P0）

1. **首页 AI 日报** — `HomeProvider.generateDailyBrief()` + `AiDailyBriefCard` 结构化卡片
2. **系统提醒闭环** — macOS 本地通知链路（数据字段已有，调度未落地）

### 中期（P1）

1. 联系人关系洞察 — AI 对联系人历史生成跟进提示与摘要
2. Outlook 日历 + 通讯录接入 — Microsoft Graph OAuth
3. 联系人详情附件模块深化
4. 节气等更多时间节点源
5. 设置中心偏好扩展（语言、通知策略、导入导出偏好）

### 长期（P2）

1. 云同步（Apple 生态用 iCloud，跨平台用自选后端）
2. Onboarding 引导流程（首次启动数据导入）
3. 系统托盘 / 菜单栏快捷捕获
4. Windows / iOS 构建与行为验证

## 当前阶段不做的事

以下方向已知有价值，但不在当前阶段主迭代范围内：

1. 多端实时协作（实时协同编辑）
2. 大规模 AI 自动化工作流（无人工确认的全自动写入）
3. 复杂附件解析流水线（OCR / 语义提取管道）
4. WidgetKit / 桌面小组件（无法用 Flutter 实现，无跨平台方案）
